Sevakam Database Structure

Main database
- Firestore = main data
- Firebase Storage = images and documents

Main collections

1. users
Path
- users/{uid}

Purpose
- Main account record for every user

Fields
- name: String
- email: String
- role: String
- photoUrl: String
- city: String
- location: String
- phoneNumber: String
- lastActiveAt: Timestamp
- createdAt: Timestamp
- updatedAt: Timestamp

Subcollections

users/{uid}/app/settings
- notifications: Map<String, dynamic>
- darkMode: Boolean

users/{uid}/chats/{chatId}
- otherUserId: String
- otherUserName: String
- otherUserPhotoUrl: String
- lastMessage: String
- lastMessageAt: Timestamp
- unreadCount: Number
- orderId: String
- userRole: String

users/{uid}/chats/{chatId}/messages/{messageId}
- senderId: String
- senderRole: String
- text: String
- imageUrl: String
- createdAt: Timestamp
- readBy: List<String>

users/{uid}/helpTickets/{ticketId}
- title: String
- status: String
- createdAt: Timestamp
- updatedAt: Timestamp
- lastMessage: String
- lastMessageAt: Timestamp
- lastMessageSenderRole: String

users/{uid}/helpTickets/{ticketId}/messages/{messageId}
- senderId: String
- senderRole: String
- text: String
- imageUrl: String
- createdAt: Timestamp

users/{uid}/addresses/{addressId}
- label: String
- address: String
- city: String
- latitude: Number
- longitude: Number
- isDefault: Boolean
- createdAt: Timestamp
- updatedAt: Timestamp

Feature map
- Auth
  - users
- Finder profile
  - users + finders
- Provider profile
  - users + providers
- Search and browse
  - categories + services + providerPosts + finderPosts
- Booking
  - orders + users/{uid}/addresses
- Chat
  - users/{uid}/chats + messages
- Support
  - users/{uid}/helpTickets + messages
- KYC
  - providers
- Subscription
  - providers
- Home promotions
  - adminPromotions

Storage folders
- avatars/
- portfolio/
- provider_kyc/
- promotions/
