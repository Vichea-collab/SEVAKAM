Sevakam Project Report

1. Project summary
- Sevakam is a multi-role service marketplace app.
- It has 3 main parts:
  - Finder app
  - Provider app
  - Admin web panel
- It also has a backend API built with Node.js and Firebase Admin.
- Targets:
  - Android
  - iOS
  - Web

2. Main tools and technologies

Frontend
- Flutter
  - Main UI framework for finder app, provider app, and admin web.
- Dart
  - Main programming language for frontend code.
- Google Fonts
  - Used for app typography.

Backend
- Node.js
  - Runtime for backend services and scripts.
- Express
  - REST API server for auth, users, chat, orders, posts, subscriptions, and admin.

Firebase
- Firebase Auth
  - Used for login and account identity.
- Cloud Firestore
  - Main database for app data.
- Firebase Storage
  - Used for image and document uploads.
- Firebase Messaging
  - Used for push notification support.
- Firebase App Check
  - Helps protect app access in production.
- Firebase Admin SDK
  - Used by backend to read/write Firebase data securely.

Maps and location
- Google Maps Flutter
  - Used for booking map picker and location UI.
- flutter_map
  - Used in map-related UI flow support.
- Geolocator
  - Used for current location and tracking.

Payments
- Stripe
  - Live subscription payment system for providers.
- bakong-khqr
  - Present in backend for KHQR-related support and future/payment-side work.

Images and media
- image_picker
  - Used to pick local images from device/admin side.
- cached_network_image
  - Used to load and cache remote images.

Local storage
- shared_preferences
  - Used for local settings, app state cache, and simple preferences.

Networking and utility
- http
  - Used for API calls.
- flutter_dotenv / dotenv
  - Used for environment config.
- url_launcher
  - Used to open external links.

UI support
- shimmer
  - Used for loading placeholders.
- lottie
  - Used for richer loading/empty/animation states.
- table_calendar
  - Used in calendar and availability-related flows.

Backend support tools
- cors
  - Enables backend cross-origin requests.
- helmet
  - Adds security headers to backend responses.
- morgan
  - Logs backend HTTP requests.
- swagger-jsdoc + swagger-ui-express
  - API documentation support.
- nodemon
  - Local backend dev auto-reload.

3. Firebase services and what they do in this project

Firebase Auth
- Login with email/password
- Google Sign-In
- Role-based access for finder, provider, and admin

Cloud Firestore
- Stores users
- Stores finder profiles
- Stores provider profiles
- Stores categories and services
- Stores finder request posts
- Stores provider public posts
- Stores orders
- Stores chats and messages
- Stores support tickets and support messages
- Stores admin promotions
- Stores admin broadcasts
- Stores promo codes
- Stores KYC and subscription-related provider data

Firebase Storage
- Stores profile avatars
- Stores provider portfolio photos
- Stores provider KYC front/back ID images
- Stores admin promotion banner images

Firebase Messaging
- Supports notification delivery for app events
- Used for chat, order, support, and admin broadcast notification flows

Firebase App Check
- Used to reduce abuse and protect access in production environments

Firebase Admin SDK
- Used by backend to manage Firebase from secure server-side code
